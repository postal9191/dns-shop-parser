# Решение проблемы: бот падает после первого запуска

## Проблема
При повторном запуске (через ~1 час) бот получал ошибки 401/403 и падал. Qrator challenge не мог быть решен.

### Причина
1. Куки Qrator (`qrator_jsid2`) протухают через несколько часов
2. При ошибке solve_qrator.js парсинг просто прерывался без повторных попыток
3. Не было механизма переинициализации кук при ошибке 401/403 во время парсинга

## Решение

### 1. Автоматические повторные попытки для Qrator (session_manager.py)
- `_resolve_qrator()` теперь пытается решить challenge **3 раза** подряд
- Между попытками 5-секундная задержка
- Если все 3 попытки неудачны → сессия не инициализируется

```python
async def _resolve_qrator(self, retry_count: int = 0, max_retries: int = 3) -> bool:
```

### 2. Проверка результата инициализации (session_manager.py)
- `_init_session()` теперь **требует успеха** Qrator challenge
- Если Qrator не решен → возвращает False и не продолжает парсинг

### 3. Автоматические повторные попытки инициализации (parser.py)
- При ошибке инициализации сессии парсер **3 раза** пытается переинициализировать
- Задержки: 10, 20, 30 секунд (экспоненциальный backoff)

```python
for attempt in range(max_init_retries):  # 3 попытки
    if await self.init_session_browserless():
        break
    wait_time = 10 * (attempt + 1)  # 10, 20, 30 сек
    await asyncio.sleep(wait_time)
```

### 4. Обработка ошибок 401/403 во время парсинга (parser.py)
- Если при получении товаров возникает `CookiesExpiredError` (401/403)
- Парсер **автоматически переинициализирует** сессию
- Затем повторно пробует получить товары

### 5. Отслеживание подряд идущих ошибок (run.py)
- Главный цикл считает количество подряд идущих ошибок
- После 5 подряд идущих ошибок выводит подробные рекомендации:
  - Проверить логи в logs/app.log
  - Проверить интернет
  - Проверить Node.js и Playwright
  - Проверить, не заблокирован ли IP на dns-shop.ru
- Затем цикл замедляется до 60 сек между попытками

## Логирование для отладки

При ошибке Qrator будут логи типа:
```
[SESSION] Решаю Qrator challenge (попытка 1/3)...
[QRATOR] Return code: 1
[SESSION] ⚠️ Не удалось решить Qrator, повторная попытка через 5 сек...
[SESSION] Решаю Qrator challenge (попытка 2/3)...
...
[SESSION] ✅ Qrator challenge решен, jsid2 добавлена
```

При ошибке инициализации будут логи типа:
```
[MAIN] Попытка инициализации сессии (1/3)
[MAIN] ⚠️ Инициализация не удалась, повторная попытка через 10 сек...
[MAIN] Попытка инициализации сессии (2/3)
...
```

При ошибке куки во время парсинга:
```
[PARSE] ⚠️ Куки устарели при получении категорий
[PARSE] Переинициализирую сессию и пытаюсь снова (2/3)...
[SESSION] Инициализация сессии (без браузера)...
[PARSE] Повторная попытка получения категорий...
```

## Тестирование

Для проверки работы можно:

1. **Первый запуск:** `python parser.py` - должен работать нормально
2. **Второй запуск через ~2 часа:** парсер должен автоматически переинициализировать куки
3. **Имитация ошибки:** можно вручную удалить куки и запустить - должен автоматически переинициализировать
