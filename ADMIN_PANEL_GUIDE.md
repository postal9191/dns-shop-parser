
- ✅ `config.py` — чтение TELEGRAM_CHAT_ADMIN из .env

## 📱 Команды бота

### /admin
Открывает админ-панель со следующими кнопками:

#### ▶️ **Запустить**
- Запускает парсер, если он остановлен
- Если уже работает → уведомление "Парсер уже работает"
- Логируется в logs/app.log

#### ⏹ **Остановить**
- Останавливает парсер до следующей команды
- Если уже остановлен → уведомление
- Главный цикл переходит в режим ожидания

#### 🔄 **Перезапустить**
- Останавливает парсер → 1 сек ожидания → запускает заново
- Полный перезапуск цикла

#### ⏱ **Интервал**
- Бот запрашивает новое значение в секундах
- Валидация: минимум 60 сек
- Применяется после следующей итерации парсера
- Пример: вводите `1800` для установки 30 минут

#### 📄 **Логи**
- Отправляет последние 100 строк из `logs/app.log`
- Если текст больше 4096 символов → разбивает на части
- Показывает актуальное состояние парсера

#### 📊 **Статус**
- Показывает:
  - ▶️ **Работает** / ⏸️ **На паузе** / ⏹️ **Остановлен**
  - 📊 **Интервал парсинга** (в сек)
  - ⏰ **Время запуска** (YYYY-MM-DD HH:MM:SS)
  - 📈 **Количество итераций** с момента запуска

- Все админ-команды проверяют `user_id == TELEGRAM_CHAT_ADMIN`

## 🏗️ Архитектура

### ParserController (`services/admin_panel.py`)
Хранит состояние парсера и управляет им:
```python
controller = ParserController()
await controller.start()      # Запуск
await controller.stop()       # Остановка
await controller.restart()    # Перезапуск
await controller.set_interval(seconds)  # Новый интервал
status = controller.get_status()  # Текущий статус
```

### Интеграция в run.py
```python
parser_controller = ParserController()
telegram_bot = init_telegram_bot(db, parser_controller)

# В main_cycle():
if not parser_controller.state.is_running:
    # ждем команду от админа
    await asyncio.sleep(5)
    continue

# Проверка нового интервала
new_interval = parser_controller.get_pending_interval()
if new_interval:
    wait_time = new_interval
```

### Интеграция в telegram_bot.py
- `handle_update()` обрабатывает `/admin` команду и callback_query
- `_handle_admin_command()` показывает inline-кнопки
- `_handle_callback_query()` обрабатывает нажатия кнопок
- `_answer_callback()` отправляет уведомления

## ⚙️ Детали реализации

### Состояние парсера (ParserState)
```python
@dataclass
class ParserState:
    is_running: bool = False        # Работает ли парсер
    is_paused: bool = False         # На паузе ли
    current_interval: int = 3600    # Текущий интервал (сек)
    last_start_time: Optional[datetime] = None  # Когда был запущен
    iteration_count: int = 0        # Количество итераций
```

### Управление через asyncio.Event
- `_stop_event` — для остановки парсера
- `_pause_event` — для паузы (зарезервировано)
- `_interval_changed_event` — для смены интервала

### Обработка ошибок
Все методы имеют try/except блоки:
```python
try:
    # действие
except Exception as exc:
    logger.error("[TG BOT] Ошибка: %s", exc)
    # отправка уведомления пользователю
```

## 🐛 Решение проблем

### Парсер не реагирует на команды
- Проверьте, что `TELEGRAM_CHAT_ADMIN` совпадает с вашим user_id
- Убедитесь, что `TELEGRAM_TOKEN` корректный
- Проверьте логи: `grep "TG BOT" logs/app.log`

### Интервал не меняется
- Новый интервал применяется ПОСЛЕ текущей итерации
- Проверьте, что введено число > 60
- Смотрите лог: `grep "Интервал изменен" logs/app.log`

### Логи не отправляются
- Убедитесь, что `logs/app.log` существует
- Проверьте права доступа к файлу логов
- Лимит Telegram: 4096 символов на сообщение
